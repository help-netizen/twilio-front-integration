# ALB-100 — журнал решений ночного автономного прогона (2026-06-12)

Все решения принимались по принципу «Recommended». Проверь и скорректируй.

1. **Кастомный /signin отложен** — вход остаётся через Keycloak hosted page.
   Причина: AuthProvider построен на keycloak-js (login-required redirect);
   кастомный вход требует ROPC (Direct Access Grants) и рефакторинга
   AuthProvider — рискованно делать без проверки вживую. /signup и /onboarding
   полностью кастомные. Кнопка Google на /signup идёт через idpHint.
2. **FEATURE_SMS_2FA по умолчанию ВЫКЛ** — код готов (cookie albusto_td,
   trusted_devices, 401 PHONE_VERIFICATION_REQUIRED), включается env-флагом
   после настройки фронтового OTP-перехвата для логина (сейчас OTP «на вход»
   используется только в онбординге; для включения на логин нужен ещё
   UI-перехват — помечено как next step).
3. **FEATURE_SELF_SIGNUP по умолчанию ВЫКЛ** — публичный /api/public/signup
   отвечает 503, пока ты не включишь (безопасный дефолт на проде).
4. **Телефонный шаг перенесён в /onboarding** (после первого логина), а не до
   него: email-верификация всё равно требует выйти в почту; так wizard короче,
   а «телефон обязателен до входа в продукт» соблюдён — без онбординга в
   продукт не попасть (OnboardingGate).
5. **CompaniesManager (существующий) оставлен на /api/admin/companies** —
   роут переведён на requirePlatformRole; новый канонический
   /api/platform/companies (PF103) реализован параллельно (list richer:
   users_count, last_activity). Перевод UI на новый API — мелкий follow-up.
6. **Sanitizer вычистил 25 legacy-нарушений** в 14 route-файлах
   (req.user.company_id → companyFilter), включая zenbooker/payments,
   messaging, portal. sessions.js — allowlist (audit-context only).
7. **HARDENING-002 покрыл calls/messaging/leads/email.**
   conversations.js оказался НЕ смонтирован (легаси) — не трогал.
   Email уже был под guards; leads уже были tenant-scoped — добавлены только
   per-action permissions.
8. **Rebranding**: видимые строки (шапка Albusto, <title>, settings-копирайт,
   API docs, SoftPhone label). CSS-токены --blanc-* и внутренние имена не
   тронуты (это не пользовательские строки).
9. **Прод НЕ деплоился ночью** — изменения только в ветке. Миграции 096/097
   проверены на копии прод-схемы (идемпотентны), но не применены.

   создать OAuth client в Google Console (redirect:

   Без него онбординг деградирует в ручной ввод города (работает).

   (сейчас не настроен → письма не уйдут). Дай SMTP-креды (или использую
