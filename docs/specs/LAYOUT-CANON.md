# LAYOUT-CANON — правила вёрстки и структуры (анти-матрёшка)

Дополнение к PALETTE-V2 и FORM-CANON. Палитра — 60% сходства с эталонным мокапом;
остальные 40% — СТРУКТУРА: отступы, вложенность, владение пространством. Этот
документ — обязательный шаблон для структурной волны (переверстка экранов после
W2) и для ЛЮБОЙ новой вёрстки с этого момента.

## Правила

1. **Каждый div обязан иметь роль.** Роль = поверхность (фон/бордер/радиус),
   layout (flex/grid с gap), скролл-контейнер или семантика. Обёртка «просто
   так» или с единственным классом-прокси — удаляется. Цель: тайл списка ≤ 5
   уровней вложенности от корня карточки, секция страницы ≤ 7.

2. **Отступы задаёт родитель, а не дети.** Ритм между сиблингами — только
   `gap` у flex/grid контейнера (или `space-y-*` как эквивалент). Никаких
   `margin` на детях для расстояний между собой: margin-стеки схлопываются,
   дублируются и дают «прыгающие» отступы — источник нынешней неровности.

3. **Padding живёт там, где поверхность.** Карточка сама владеет своим
   внутренним отступом (`p-*` на элементе с фоном/бордером). Контент внутри НЕ
   добавляет внешних полей «на всякий случай». Страница владеет полями страницы
   (один wrapper с max-width+padding), секции — только `gap` между собой.

4. **Один скролл-контейнер на область.** Прокручивается либо страница, либо
   явная панель — вложенные «скролл внутри скролла» запрещены (кроме канонных
   DialogBody / боковых списков).

5. **Ритм фиксированный, из шкалы.** Между секциями страницы — 24 (space-y-6);
   внутри секции между группами — 14 (gap-3.5); внутри карточки — 8–12.
   Промежуточные «на глаз» значения (13px, 18px, 22px...) не появляются.

6. **Повтор ≥3 раз = компонент.** Одинаковая связка обёрток, повторённая в
   файле или между файлами (карточка списка, строка-метаданные, пилл статуса,
   скелетон) — извлекается в shared/ (MobileListCard, StatusPill, SkeletonLine
   — из аудита), а не копируется.

7. **Контейнер невидим; поверхность — только у контента.** (Прямое требование
   владельца, 2026-07-02, на примере Pulse.) Обёртка списка/области НЕ имеет
   визуальных признаков: ни фона, ни бордера, ни скруглений, ни тени — она
   только раскладывает (layout-only). Белую поверхность несут САМИ контентные
   юниты — тайл разговора, карточка работы, секция формы. Список = тайлы прямо
   на канвасе с воздухом между ними (gap 8–12), не «панель с рядами внутри».
   Уровней поверхности над канвасом — ровно один (канвас → тайл); «карточка в
   карточке» допустима только у канонных оверлеев (шторка, шит). Всё
   декоративное (лишние подложки, рамки, скругления обёрток) — под нож;
   остаются только элементы, несущие информацию или действие.

8. **Сетки — grid, а не флекс-акробатика.** Двухколоночные пары полей и
   карточные раскладки — `grid grid-cols-* gap-*`; flex — для строк
   (иконка+текст+действие), не для сеток.

## Мерило (проверяется на каждой структурной волне)

- **DOM-глубина ключевых элементов** (снимается в браузере одной командой:
  `document.querySelector('<тайл>')` → глубина до листа): тайл Jobs/Leads/Pulse
  ≤ 5, секция настроек ≤ 7. До/после фиксируется в PR-описании волны.
- **Число DOM-узлов страницы** на фиксированных роутах UI-BASELINE-ROUTES —
  не растёт, на переверстанных экранах падает ≥20%.
- Скриншот-протокол W0 действует как и раньше.

## Порядок структурной переверстки (после W2-палитры)

По одному экрану за подход, каждый = отдельный коммит со скриншотами и
метрикой глубины: (1) Settings-шелл + худшие разделы (из аудита), (2) карточки
списков Jobs/Leads/Tasks (+shared-компоненты п.6), (3) панели
Estimate/Invoice, (4) Schedule, (5) Pulse-зона — последней, поштучно (FREEZE
до этого момента). Правило волны прежнее: в одном коммите либо структура,
либо поведение — никогда вместе.
