# UI-BASELINE-ROUTES — фиксированный чек-лист роутов для скриншот-контроля

Волна W0 плана `UI-AUDIT-001.md`. Перед КАЖДОЙ волной рефактора (и после неё) снимаем одни и те же роуты в одних и тех же вьюпортах и сравниваем глазами. Снимки лежат в `docs/ui-baseline/<дата>-<волна>/` (в git НЕ коммитятся — папка в .gitignore; git хранит только этот список).

Вьюпорты: **desktop 1440×900**, **mobile 390×844** (только помеченные 📱).

Интерактивные состояния (снимаются вручную на 01–02 при палитро-волнах W1/W2):
- A. `/pulse` — открытая карточка контакта (FloatingDetailPanel, z-80)
- B. `/jobs` — открытая карточка job + модалка reschedule поверх (z-140 поверх z-80 — регресс-контроль LAYER-Z-FIX)
- C. `/leads` — форма нового лида (Dialog variant="panel»: floating-поля)
- D. мобильный BottomSheet (гео-меню view-options на /jobs 📱)

Правило волны: `npm run build` зелёный → снимки → сравнение с базой → деплой. Для W1 (токенизация) эталон = «визуально ноль» на канонических страницах; допустимые отличия — только в заранее объявленных местах-нарушителях.
