# PULSE-PLAYER-001 — плавающий плеер записей звонков в Pulse (OB-13)

**Дата:** 2026-07-19 · **Статус:** спека утверждена интервью владельца, реализация Claude (без GPT — прямое указание)
**Владелец сказал:** «Это должен быть плеер на странице в виде hover-бара поверх остального, с нормальными размерами, а на карточке звонка фронт станет чище и свободнее. Не нужно продолжать играть плеер если переходишь на другую страницу, только в Pulse». Дополнения: «только текущий функционал, ничего лишнего» (выбранный минимум = перемотка + скорость; БЕЗ download / waveform / запоминания скорости / глобального плеера).

## Проблема

Плеер записей живёт ВНУТРИ карточки звонка (`PulseCallAudioPlayer`): кнопочный ряд play/±10s/время занимает место на каждой карточке, при этом это НЕ полноценный плеер — нет полосы прогресса с перемоткой мышью/пальцем, элементы мелкие. При скролле таймлайна играющий звонок «уезжает» вместе с карточкой, управление пропадает из вида.

## Решение (UX)

1. **Плавающий бар-плеер** поверх контента Pulse — `position: fixed`, низ по центру, ширина `min(680px, 100vw−32px)`, высота ~64px, поверхность frosted-glass (`--blanc-surface` + blur), радиус 16px, тонкая рамка `--blanc-line`, мягкая тень (плавающий слой). z-слой **70** — ниже `OVERLAY_Z.panel` (80): любые панели/диалоги перекрывают плеер.
2. **Состав бара** (слева направо): Play/Pause (основная кнопка 40px, акцент `--blanc-accent`), −10s, +10s, блок подписи (телефон/имя + время звонка, одна строка, truncate), полоса прогресса (`input[type=range]`, flex-1, перемотка драгом/кликом/клавиатурой), время `тек / общ` (mono), чип скорости (цикл 1×→1.25×→1.5×→2×), ✕ (стоп + скрыть бар).
3. **Карточка звонка становится чище:** кнопочный ряд плеера уходит из карточки. Остаётся один компактный ряд: Play/Pause-чип («Play recording» / Pause, показывает состояние ИМЕННО этого звонка), тумблеры Summary / Transcript, сентимент-эмодзи. Панели Summary/Transcript/Entities не меняются функционально.
4. **Seek из контента остаётся:** клик по строке транскрипта с таймстемпом и по entity с `start_ms` перематывает ОБЩИЙ плеер (если играет другой звонок — бар переключается на этот и играет с нужной секунды).
5. **Только Pulse:** провайдер и бар смонтированы внутри `PulsePage` (все /pulse-маршруты рендерят её одну) → уход на любую другую страницу размонтирует `<audio>` и звук гарантированно останавливается. Внутри Pulse (список ↔ таймлайны) плеер живёт и продолжает играть.
6. **Мобайл:** тот же бар, прижат к низу с учётом `env(safe-area-inset-bottom)`; высота тапов ≥40px. Никакого BottomSheet — это не блокирующий оверлей.
7. **Live-транскрипция** (звонок без audioUrl) — путь не меняется.
8. Страница `/calls` (легаси `CallAudioPlayer`) — ВНЕ скоупа (владелец: только Pulse).

## Архитектура

- **`frontend/src/components/pulse/pulsePlayer.tsx`** — контекст + провайдер + хук:
  - Единственный `<audio>` элемент живёт в провайдере (src = `audioUrl?token=`, как сейчас).
  - Состояние: `track {callSid, audioUrl, label, durationHint} | null`, `isPlaying`, `currentTime`, `duration`, `rate`.
  - API: `playTrack(t)` (тот же callSid → toggle, иной → загрузить и играть), `toggle()`, `seekTo(sec)`, `seekTrack(t, sec)`, `skip(±s)`, `cycleRate()`, `close()`.
  - Дефолт контекста — инертная заглушка (`track:null`, no-op), чтобы рендер карточки вне провайдера (тесты) не падал.
- **`frontend/src/components/pulse/PulsePlayerBar.tsx`** — бар; `null` без трека.
- **`PulseCallAudioPlayer.tsx`** — минус локальный `<audio>` и стейт плеера; чип и seek'и через хук. Подпись трека собирает карточка: `formatPhoneDisplay(otherParty) · h:mm AM/PM`.
- **`PulsePage.tsx`** — обернуть возврат в `<PulsePlayerProvider>`, рядом с диалогами отрендерить `<PulsePlayerBar />`.
- Бэкенд не трогаем. Миграций нет. Ни одного нового запроса к API.

## Тест-кейсы (vitest, frontend)

- TC-PP-01: без трека бар не рендерится.
- TC-PP-02: `playTrack` → бар виден, подпись видна, `audio.play()` вызван.
- TC-PP-03: повторный `playTrack` того же callSid → pause (toggle).
- TC-PP-04: `playTrack` другого callSid → src сменился, играет новый.
- TC-PP-05: `seekTrack(другой звонок, 42s)` → трек переключён, `currentTime=42`, играет.
- TC-PP-06: ✕ → `audio.pause()`, трек сброшен, бар скрыт.
- TC-PP-07: `cycleRate` → 1→1.25→1.5→2→1, `audio.playbackRate` применён.
- TC-PP-08: размонтирование провайдера (уход из Pulse) → pause вызван (звук не продолжается).
- TC-PP-09: карточка: чип показывает Pause только когда активный трек = её callSid и isPlaying.
- Существующие сьюты pulse* не краснеют; `npm run build` зелёный.

## Границы / НЕ делаем

Нет: download, waveform, запоминание скорости, продолжение между страницами, MediaSession/фоновое воспроизведение, изменения `/calls`, изменения бэкенда.
