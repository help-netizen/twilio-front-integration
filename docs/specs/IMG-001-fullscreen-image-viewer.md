# Спецификация: IMG-001 — Fullscreen Image Viewer

## Общее описание

Shared-компонент полноэкранного просмотра изображений (lightbox). Открывается при клике на превью в галерее вложений. Позволяет рассмотреть мелкие детали (серийные номера, чеки), навигировать между картинками, поворачивать. UX аналогичен Telegram.

---

## Сценарии поведения

### Сценарий 1: Открытие полноэкранного просмотра

- **Предусловия:** Открыта панель с AttachmentsSection, есть хотя бы 1 изображение
- **Шаги:**
  1. Пользователь кликает на область превью изображения (тёмный блок с картинкой)
  2. Frontend рендерит `FullscreenImageViewer` — fixed overlay на весь экран
  3. Body scroll блокируется (`overflow: hidden`)
  4. Текущее изображение отображается на максимальный размер (max-height: 85vh)
- **Ожидаемый результат:** Полноэкранный оверлей с крупным изображением, затемнённый фон
- **Побочные эффекты:** нет (frontend-only)

### Сценарий 2: Навигация между изображениями

- **Предусловия:** Fullscreen viewer открыт, в галерее > 1 вложения
- **Шаги:**
  1. Пользователь нажимает стрелку вправо (кнопка или клавиша →)
  2. Показывается следующее изображение, rotation сбрасывается в 0
  3. Миниатюры внизу обновляют выделение
- **Ожидаемый результат:** Плавная смена изображения
- **Граничные случаи:**
  - На первом изображении кнопка ← disabled
  - На последнем изображении кнопка → disabled
  - Non-image вложения пропускаются в strip миниатюр, но index указывает на общий массив attachments

### Сценарий 3: Поворот изображения

- **Предусловия:** Fullscreen viewer открыт с изображением
- **Шаги:**
  1. Пользователь нажимает кнопку поворота (RotateCcw)
  2. Изображение поворачивается на -90° через `RotatableImage`
  3. При 90°/270° применяется scale-компенсация
- **Ожидаемый результат:** Картинка повёрнута, вписана в viewport

### Сценарий 4: Закрытие viewer

- **Предусловия:** Fullscreen viewer открыт
- **Триггеры:**
  - Нажатие Escape
  - Клик по затемнённому фону (мимо картинки)
  - Клик на кнопку X
- **Ожидаемый результат:** Оверлей закрывается, body scroll восстанавливается

### Сценарий 5: Открытие оригинала

- **Предусловия:** Fullscreen viewer открыт
- **Шаги:** Пользователь кликает иконку ExternalLink
- **Ожидаемый результат:** URL изображения открывается в новой вкладке

---

## Граничные случаи

1. Единственное изображение → кнопки навигации disabled, strip миниатюр скрыт
2. Все вложения — файлы (не images) → fullscreen не открывается (нет cursor: zoom-in)
3. Изображение не загрузилось (broken URL) → показывается пустой контейнер, кнопки навигации работают
4. Множественные быстрые клики навигации → rotation сбрасывается при каждом переключении

## Обработка ошибок

1. Ошибка загрузки изображения → img onError можно обработать, но не критично (broken image icon)
2. z-index конфликт с другими overlays → использовать z-[9999]

## Взаимодействие компонентов

- `AttachmentsSection` (PaymentDetailPanel) → state `fullscreen` → `FullscreenImageViewer`
- `FullscreenImageViewer` переиспользует `RotatableImage` с prop `fullscreen=true`
- State `galleryIndex`, `rotation` расшарены между inline preview и fullscreen viewer

## API-контракты

Не применимо — frontend-only компонент, данные уже загружены.

## Безопасность

Не применимо — компонент не делает сетевых запросов.
